## 3. AI Coding が効く領域

組み込みシミュレーションの互換 runtime は、製品そのものではないため成果が見えにくい一方、壊れると開発全体を止めます。人間にとっては、ioctl ABI の調査、stub 実装、ログ確認、回帰テスト、ドキュメント更新が続く、モチベーションを保ちにくい仕事です。

AI エージェントにとっては、この種の作業は相性が良い領域です。

- `strace` で実際に使う `open/ioctl/read/write` を抽出する
- 既存の調査ログや小さな stub から fake device runtime へロジックを移植する
- プロジェクトが使う ABI の最小範囲だけを実装する
- EC2 と実機の両方で同じ target manifest を検証する
- 壊れたときにログと状態確認から再修正する

つまり Gapless Agent Runtime は、単にクラウドで組み込みアプリを動かすための環境ではなく、**人手では費用対効果が合わず腐りやすい互換 runtime を、AI が継続的に維持するための環境**です。
