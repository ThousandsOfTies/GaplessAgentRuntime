## なぜ今の PoC 資産がそのまま土台になるか

| 既存資産 | この構想での役割 |
|---|---|
| `gar-tools/cuse-stubs/`（cuse_gpio / cuse_i2c / cuse_spi） | **存在しないデバイスを OS に生やす捏造装置** |
| `vl53l0x_sim.c` / `mfrc522_sim.c` / `ssd1306_sim.c` | デバイスの振る舞いをコードで定義 ＝ 新デバイスも同じ枠で「発明」できる |
| `web-bridge/` | 人間が観察・介入する窓（コックピット） |
| `scenarios/*.json` | ループの初期条件を宣言的に与える仕組み |

→ 「架空デバイスを 1 個 `.c` で書く → CUSE で OS に生やす → エージェントが get/put で叩いて
タスクさせる → 上手くいかない／面白くないを FB して仕様を進化させる」という閉ループが、
今の構造の延長で組める。
