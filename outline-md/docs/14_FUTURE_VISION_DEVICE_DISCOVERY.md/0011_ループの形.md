## ループの形

```
架空デバイス仕様 ──→ CUSE で仮想デバイス生成 ──→ タスク sim（エージェントが put/get）
      ↑                                                      │
      └──────────── FB（面白い? 強い? 物理的に妥当?）←───────┘
                              │
                      収束 ──→ 現実への提案「このデバイス作って！」
```

評価軸（D）を何にするかで研究の色が決まる:
1. **ゲーム性** … タスク／対戦が面白いか → エンタメ・ゲーム AI
2. **物理的実現可能性** … その架空デバイス、現実に作れるか → ここを入れると「妄想」が「提案」に化ける
3. **学習効率** … その新デバイスがあると学習が速いか → ロボ・強化学習
