## 6. 移行ステップ

gar-tools リポジトリと Gapless Agent Runtime リポジトリで分担。各ステップの完了は `gar sim env diag --json` の `ok` と、`gar sim ui button/rfid`、`gar sim env status --json`、`gar sim env log` による実機能確認で判定する。

| Step | 対象リポ | 内容 | 完了条件 |
|---|---|---|---|
| **S1/S2** | gar-tools | `cuse_gpio` プロトタイプ調査。fd 払い出し問題を確認。 | `gpio-sim` ルートへ方針転換 |
| **S3** | Gapless Agent Runtime | `gar sim env start` / 起動手順から `LD_PRELOAD=gpio_shim.so` を削除。GPIO は `gpio-sim` へ移行。 | sensor_demo を `LD_PRELOAD` なしで起動 → GPIO 動作 |
| **S4** | gar-tools | `cuse_spi` 実装。MFRC-522 register sim を移植。 | RFID Tap が OLED / ログに UID 反映 |
| **S5** | Gapless Agent Runtime | `LD_PRELOAD=spi_shim.so` を削除。`cuse_spi` を起動。 | `LD_PRELOAD` 完全廃止、起動コマンド `~/sensor_demo` のみ |
| **S6** | Gapless Agent Runtime | docs (01, 03, 05, 06, AGENT.md, README.md) の記述を移行完了状態に更新。`gpio_shim.so` / `spi_shim.so` を artifact 一覧から削除。 | 完了 |
| **S7** | 両リポ | 旧 `gpio_shim` / `spi_shim` のソースを削除。 | git history で参照可能 |

S1〜S2 で GPIO 経路の正しさを早期検証してから SPI に進む順番が安全。S3 と S4 は並列可。

2026-06-03 の確認により、GPIO は S1/S2 の CUSE ルートではなく `gpio-sim` + GPIO chardev v2 ルートを優先する。暫定の置き換えステップ:

| Step | 対象リポ | 内容 | 完了条件 |
|---|---|---|---|
| **G1** | Gapless Agent Runtime | `gar sim env gpio-sim-check --json` で EC2 の `gpio-sim` 対応を診断する。 | 完了 |
| **G2** | embedded-poc-app | `sensor_demo.c` の GPIO helper を v1 (`GPIO_GET_LINEHANDLE_IOCTL`) から v2 (`GPIO_V2_GET_LINE_IOCTL`) へ移行する。アプリは引き続き `/dev/gpiochip0` だけを開く。 | 完了 |
| **G3** | gar-tools / Gapless Agent Runtime | bridge.py が gpio-sim の sysfs (`/sys/devices/platform/.../sim_gpioN/{pull,value}`) と同期する。 | 完了 |
| **G4** | Gapless Agent Runtime | `gar sim env start` に gpio-sim setup を組み込み、GPIO の `LD_PRELOAD` を削除する。 | 完了 |

G2 確認結果 (2026-06-03):

- `sensor_demo.c` を GPIO chardev v2 に移行。
- Codespaces (`gar-build-env`) で `aarch64-linux-gnu-gcc` ビルド成功。
- アプリに simulation 固有の chip 選択は入れない。`/dev/gpiochip0` を開くままにし、simulation runtime 側で gpio-sim を `/dev/gpiochip0` として提供する。
- `gar sim env start` で gpio-sim chip を作成し、必要なら `/dev/gpiochip1` を `/dev/gpiochip0` に bind mount する。
- EC2 で環境変数なしの `~/sensor_demo` が GPIO setup 成功。
- `gar sim ui button press 17` で `[btn] System ON/OFF` を確認。
- 実行中の `sim_gpio18/value` と panel state の LED18 が同期することを確認。
- `gar sim ui rfid tap 04:AB:CD:EF:01:23` で UID ログを確認し、`gar sim ui rfid remove` で `present=0` に復帰。
