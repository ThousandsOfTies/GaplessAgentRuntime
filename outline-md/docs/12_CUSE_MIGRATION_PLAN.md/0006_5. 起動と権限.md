## 5. 起動と権限

### 5.1 systemd unit 化

EC2 上の simulation runtime は systemd で常駐させる形へ移行済み。`gar sim env start` は `hardware/*.csv` から GPIO line と I2C/SPI dev を読み、unit / helper script を配置して `gar-sim.target` を起動する。`gar sim env stop` は同じCSV由来の service 群を停止する。

現在配置する主な unit / script:

```text
/usr/local/bin/gar-gpio-sim-start
/usr/local/bin/gar-gpio-sim-stop
/etc/systemd/system/gar-sim.target
/etc/systemd/system/gar-gpio-sim.service
/etc/systemd/system/gar-bridge.service
/etc/systemd/system/gar-cuse-i2c@.service
/etc/systemd/system/gar-cuse-spi@.service
```

EC2 動作確認済み:

```bash
systemctl is-active \
  gar-gpio-sim.service \
  gar-bridge.service \
  gar-cuse-i2c@i2c-1.service \
  gar-cuse-spi@spidev0.0.service

gar sim env diag --json --host vibecode-graviton
```

確認結果: 4 service active、`/dev/i2c-1` / `/dev/gpiochip0` / `/dev/spidev0.0` あり、`diag --json` は `ok: true`。`~/sensor_demo` 起動後、Button17 で `[btn] System ON`、RFID tap で UID ログ / OLED framebuf / bridge state 更新まで確認。

現在の unit は `$HOME` 配下の `~/cuse_i2c`、`~/cuse_spi`、`~/web-bridge` 参照から移行済み。`gar sim env deploy` は旧 artifact manifest の `~/cuse_*` / `~/web-bridge` dest も互換マップし、runtime を `/etc/gar/hardware/`、`/usr/local/sbin/`、`/usr/local/lib/gar/`、`/run/gar/` へ配置する。

現在配置する主な runtime path:

```text
/etc/gar/hardware/*.csv
/usr/local/sbin/gar-gpio-sim-start
/usr/local/sbin/gar-gpio-sim-stop
/usr/local/sbin/gar-bridge-start
/usr/local/sbin/cuse_i2c
/usr/local/sbin/cuse_spi
/usr/local/lib/gar/web-bridge/
/run/gar/
/run/gar/hw_sim.sock
```

旧計画時点の unit イメージ:

```ini
# /etc/systemd/system/gar-bridge.service
[Service]
ExecStart=/home/ubuntu/venv/bin/python3 /home/ubuntu/web-bridge/bridge.py
Restart=on-failure
User=ubuntu

# /etc/systemd/system/gar-cuse-i2c.service
[Service]
ExecStart=/home/ubuntu/cuse_i2c -f --devname=i2c-1
Restart=on-failure

# /etc/systemd/system/gar-cuse-spi.service
[Service]
ExecStart=/home/ubuntu/cuse_spi -f --devname=spidev0.0
Restart=on-failure
```

`gar sim env start / stop` は systemd unit を `systemctl start/stop` する形に切り替え済み（旧 `setsid nohup` ベースは fallback 掃除のみ残す）。

### 5.2 デバイスノードのパーミッション

CUSE が払い出すノードは root:root mode 0660 が既定。アプリは ubuntu user で動かすので、各 unit の `ExecStartPost=` で `chmod 666` する、または udev rule を置く:

```bash
# /etc/udev/rules.d/99-gar-cuse.rules
KERNEL=="i2c-1",     MODE="0666"
KERNEL=="spidev0.0", MODE="0666"
```
