# 製品の核 — 何を売るのか、誰のどの痛みに刺さるのか

> このドキュメントは「思想（Why）」でも「使い方（How）」でもなく、**製品の輪郭（What / 誰に）** を残すためのもの。
> [00_ESSENCE.md](00_ESSENCE.md) が「本質は環境横断の連続化」と抽象で語った先の、
> **「具体的に誰のどの痛みに、いくらで刺さるのか」** を地に足のついた言葉で固定する。
> 営業資料の原液。熱が冷める前に、現場の手触りで書いておく。

## 一言で

**売り物は「シミュレーション環境」でも「実機デプロイ」でもない。**
**売り物は「dev → VM 検証 → 実機 のループ全体を、AI エージェントが 1 セッションで走り切れるように、**
**環境を能力として束ねた "操作面の定義（`gar` 契約）"」である。**

実装（Linux / Graviton / CUSE / gpio-sim）は **差し替え可能な部品**であって、製品の本体ではない。
MCU・ベアメタルが相手なら VM 層を Renode に差し替える。**What は操作面、How が実装**。
（この What/How 分離は [00_ESSENCE.md](00_ESSENCE.md) の「3 にもドメインにも必然性はない」と同じ主張の、製品側の言い換え。）

## 製品が消す「2 つの往復」

Gapless Agent Runtime が顧客から取り除くのは、組み込み開発者が毎日払っている **2 種類の往復コスト**。
ここが「あったら便利」ではなく **「証明済みの痛み」** である点が、収益化の土台になる。

### 往復①：物理の往復（ROM 焼きループ）

```
PC で開発 → ビルド → ROM イメージを USB に保存 → ターゲットに挿す
→ 焼き込み → 電源・リセット → 動作確認 → 不具合 → PC に戻る → 最初へ
```

組み込みエンジニアが日常的に回しているこのループには、4 つのコストが埋まっている。

| 工程 | 失われているもの |
|---|---|
| USB 保存 → 挿す → 焼く | **物理作業の手数**（1 回数分でも、日に何十回で効く） |
| 焼いて初めて分かる | **フィードバックの遅さ**（VM で先に潰せれば、焼く回数そのものが減る） |
| 実機の前に座る必要 | **物理的拘束**（離席できない・リモート不可） |
| 不具合 → 原因 → 修正の往復 | **文脈の載せ替え**（人間がツールと環境を行き来する） |

GAR はここに二段で効く。
**(a) 焼く前に VM で同一挙動を回して「焼く回数」を減らす**、
**(b) 焼く・動かす・観察するを AI が代行して「人間を物理から解放する」**。

### 往復②：認知の往復（ログ解析ループ）

```
実機/VM で動かす → ログを取る → VSCode に貼り付ける → AI に解析させる
→ 原因の見立て → 修正 → また動かす → またログを取る …
```

これは AI エージェント時代に **全エンジニアが新しく抱え始めた往復**。
「ログを取って、貼って、解析させる」という手作業の橋渡しが、思考のリズムを刻んで分断する。

GAR では、この橋渡しを人間が行わない。**AI が取得 → 解析 → 次の手までを 1 セッションで回す**（実証済み）。
物理の往復が「身体の痛み」なら、ログ解析の往復は **「認知の痛み」**。GAR は両方を畳む。

## なぜここが「お金になる」と言えるのか

| 論点 | 評価 |
|---|---|
| 痛みは実在するか | **YES（証明済み）**。ROM 焼き往復もログ解析往復も、開発者が現に毎日時間を溶かしている |
| 痛む人は多いか | **巨大**。組み込み開発者のほぼ全域。Renode 差し替えで MCU 現場まで届く |
| 技術的に防御できるか | **NO**。公知機能の組み合わせ（[00_ESSENCE.md](00_ESSENCE.md) 自身が認めている） |
| では何で防御するか | **束ね方の設計**＝ターゲット非依存の操作面（`gar` 契約）と、AI による観察・解析の自動化という**運用資産** |

> **部品（CUSE 等の実装）には値札が付かない。**
> **だが「ROM 焼きとログ解析の往復を、ターゲット非依存で AI に畳ませる操作面」には値札が付く。**

## 競合がどこで止まっているか（空白マップ）

守備範囲を正確に切ると、既存プレイヤーは「ループのどこか一点」を埋めているだけで、
**ループ全体を 1 本に束ね、しかも観察・解析まで人間から外した者はいない**。

| プレイヤー | 守備範囲 | どこで止まっているか |
|---|---|---|
| **Wokwi** | MCU/チップ本体のエミュレーション（命令レベル） | VM の中で完結。実機・ログ解析まで束ねない |
| **Renode** | MCU/SoC のシステムエミュレーション、CI | 同上。VM 側の一点。最後は人間がログを読む |
| **QEMU** | CPU エミュレーション | 速度・実機パリティが弱い。ループの一部品 |
| **balena / Foundries.io / Torizon** | 実機フリートのデプロイ・運用 | 実機側の一点。dev→VM の連続化は守備範囲外 |
| **SOAFEE / SDV** | 「クラウド↔エッジの環境パリティ」という思想・旗 | 思想止まり。実装は各社任せ |
| **Gapless Agent Runtime** | **dev → VM → 実機 を同一操作面で連続化 ＋ 観察/解析を AI が自動化** | ← ここが空白。誰も束ねていない |

ポイントは 2 つ。

1. **横の空白**：Wokwi（VM）・balena（実機）・SOAFEE（思想）はそれぞれ「ループの一点」を持つが、**3 つを跨いで切れ目を消す統合体験**を商品として持つ者はいない。
2. **縦の空白**：どのツールも **最後は人間がログを見るところで止まっている**。GAR だけがその先（取得→解析→次の手）に踏み込んでいる。

## それでも冷静に置いておく論点

熱に流されないために、弱点も同じ強さで残す。

1. **「無い」には 2 種類ある。**
   - A：誰もやっていない（先行者利益）
   - B：やる価値が市場として証明されていない（未成立リスク）
   ROM 焼き往復という痛みが証明済みである以上、この谷間は **A 寄り**。ただし「統合体験に財布を開く」行動様式が市場に根付くには時間がかかり得る。

2. **防御は技術ではなく運用資産に宿る。**
   実装は誰でも真似できる。守れるのは、ターゲットを能力として束ねる抽象（`gar` 契約・診断・シナリオ・AI による解析ループ）を**ドメインに深く刺して積み上げた厚み**だけ。

3. **最初に深く刺すドメインを 1 つ選ぶ。**
   「何でも繋がる」は諸刃。汎用性を主張する権利は、組み込み PoC の次に **別ドメインを 1 つやり切る**ことで初めて得られる（[00_ESSENCE.md](00_ESSENCE.md) と同じ戒め）。
   そして最有力の「2 件目」は、抽象的な新領域ではなく **書き手自身の現場（ROM を焼く組み込み開発）** かもしれない。最高の検証相手は、毎日その痛みを感じている当事者である。

## だから、この一枚の意味

[00_ESSENCE.md](00_ESSENCE.md) は「本質は環境横断の連続化」と **Why** を立てた。
[03_FUTURE_VISION.md](03_FUTURE_VISION.md) は宇宙・ロボットへの **遠い射程**を描いた。
このドキュメントはその間を埋める——**「いま、誰の、どの往復を消すと、お金になるのか」** という製品の輪郭。

3 層 PoC は出発点。だが最初に値札が付くのは、遠い宇宙ではなく、
**今日も ROM を USB で焼いている開発者の、その手元の往復**である。
